1. If the pattern starts with a / then it is anchored to a particular spot in the hierarchy of files, otherwise it is matched against the end of the pathname.

  2. If the pattern ends with a / then it will only match a directory, not a regular file, symlink, or device.

  3. syncBackup chooses between doing a simple string match and wildcard matching by checking if the pattern contains one of these three wildcard characters: '*', '?', and '[' .

  • '*' matches any path component, but it stops at slashes.

  • '**' matches anything, including slashes.

  • '?' matches any character except a slash (/).

  • '[' introduces a character class, such as [a-z] or [[:alpha:]].


Here are some examples of exclude/include matching:

  • exclude *.o would exclude all names matching *.o

  • exclude /foo would exclude a file (or directory) named foo in the transfer-root directory

  • exclude foo/ would exclude any directory named foo

  • exclude /foo/*/bar would exclude any file named bar which is at two levels below a directory named foo in the transfer-root directory

  • exclude /foo/**/bar would exclude any file named bar two or more levels below a directory named foo in the transfer-root directory

  • The combination of include */, include *.c, and exclude * would include all directories and C source files but nothing else

  • The combination of include foo/, include foo/bar.c, and exclude * would include only the foo directory and foo/bar.c
